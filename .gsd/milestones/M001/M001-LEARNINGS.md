# M001 Learnings

## Decisions

- **Full ordering schema upfront (4 models + 3 enums) rather than incremental** — Defined MenuItem, Table, Order, OrderItem with Category, OrderStatus, ItemStatus enums all in M001, even though Order/OrderItem aren't used until M002. One clean migration covers everything; downstream milestones just consume.
  Source: S02-SUMMARY.md/What Happened

- **PDFKit + qrcode for QR PDF generation (D004)** — Pure Node.js pipeline: qrcode.toBuffer() for PNG → PDFKit for A4 layout. Rejected jspdf (browser-first) and @react-pdf/renderer (heavyweight). Both packages are offline-compatible per R006.
  Source: S03-SUMMARY.md/Key decisions

- **Edge-compatible cookie auth for admin (D005)** — ADMIN_PASSWORD env var + httpOnly cookie. Middleware on Edge runtime protects /admin/* routes. Proportionate to local-network threat model.
  Source: S03-SUMMARY.md/Key decisions

- **@@map() on all Prisma models** — Essential when model names collide with SQL reserved words (Order → orders, Table → tables). Applied to all 4 models proactively.
  Source: S02-SUMMARY.md/Key decisions

- **Int prices for VND** — Vietnamese đồng has no decimal usage, so Int type is simpler and more correct than Float/Decimal for all price fields.
  Source: S02-SUMMARY.md/Key decisions

## Lessons

- **create-next-app rejects capital-letter directory names** — TraSua fails validation. Workaround: scaffold into temp lowercase subdirectory, copy files up, fix package.json name field.
  Source: S01-SUMMARY.md/Deviations

- **Prisma 7 introduces prisma.config.ts** — Seed command configuration goes in the migrations block of prisma.config.ts, not in package.json scripts. The file is auto-generated by `prisma init` in v7.
  Source: S01-SUMMARY.md/Key decisions

- **Scripts outside Next.js can't use @/ path alias** — seed.ts and other standalone scripts must use relative imports (e.g. '../generated/prisma/client') because they're executed via tsx, not within Next.js's module resolution.
  Source: S02-SUMMARY.md/Patterns established

- **Next.js Route Handlers default to Edge runtime** — Must set explicit `runtime = 'nodejs'` when using Node.js APIs like pdfkit's Buffer/fs. Otherwise the route silently fails or throws at runtime.
  Source: S03-SUMMARY.md/Key decisions

## Patterns

- **PrismaClient singleton via globalThis** — src/lib/prisma.ts caches the client on globalThis in non-production to survive HMR reloads. All code imports from `@/lib/prisma` — never instantiates PrismaClient directly.
  Source: S01-SUMMARY.md/Patterns established

- **PDFKit buffer-collection for Next.js** — Collect chunks in array, await 'end' promise, then Buffer.concat() and pass to new Response(). pipe() is incompatible with Next.js Response constructor.
  Source: S03-SUMMARY.md/Patterns established

- **Prisma seed idempotency via deleteMany → recreate** — deleteMany in reverse FK order (orderItem → order → menuItem → table) then Promise.all create. Simpler than upsert when seed data doesn't need ID stability.
  Source: S02-SUMMARY.md/Patterns established

- **Cookie-based admin auth with Edge middleware** — src/middleware.ts protects /admin/* and /api/admin/* via config.matcher, exempting login routes. Cookie helpers in src/lib/auth.ts are Edge-compatible (no Node.js APIs).
  Source: S03-SUMMARY.md/Patterns established

## Surprises

- **Prisma 7 config restructuring** — prisma.config.ts was unexpected from `prisma init`. Seed command, datasource URL, and other config moved from package.json/schema.prisma into this new file. Required plan adaptation.
  Source: S01-SUMMARY.md/Deviations

- **GitHub raw URLs redirect to HTML for large repos** — Attempted to download Inter TTF from GitHub raw content URL, but large repos redirect to an HTML page instead. Google Fonts static CDN (gstatic.com) is the reliable alternative.
  Source: S03-SUMMARY.md/Key decisions

- **Deleting old migrations cleaner than incremental** — When S02 replaced the placeholder Category-only schema with 4 models, attempting incremental migration from the placeholder was problematic. Deleting the old migrations directory entirely and running a fresh `migrate dev` was cleaner and more reliable.
  Source: S02-SUMMARY.md/Deviations
