# T01: Added kitchen page, overview page, and staff layout with bottom navigation tabs for all three stations

**Added kitchen page, overview page, and staff layout with bottom navigation tabs for all three stations**

## What Happened

Created three files to complete the staff station page topology:

1. **Kitchen page** (`src/app/staff/kitchen/page.tsx`) — Thin Server Component wrapping `StationView` with `station="kitchen"`, matching the bar page pattern exactly. Vietnamese metadata: "TraSua - Bếp".

2. **Overview page** (`src/app/staff/page.tsx`) — Server Component wrapping `StationView` with `station="all"`. Vietnamese metadata: "TraSua - Tổng quan".

3. **Staff layout** (`src/app/staff/layout.tsx`) — Clean Server Component that wraps page content with bottom padding and mounts `StaffNav`.

4. **StaffNav client component** (`src/app/staff/StaffNav.tsx`) — `'use client'` component using `usePathname()` for active state detection. Three links: Quầy Bar → `/staff/bar`, Bếp → `/staff/kitchen`, Tổng quan → `/staff`. The overview link uses exact-match (`pathname === '/staff'`) to avoid false positives since all staff routes start with `/staff`. Active tab gets an amber indicator pill at top + darker text. Touch targets are `min-h-[56px] min-w-[72px]` (exceeds 44px requirement). Bottom-fixed with frosted glass backdrop (`backdrop-blur-xl`) consistent with the warm amber design system.

Design decision: Used bottom navigation (mobile-first) instead of top tabs because staff use tablets — bottom nav is more thumb-reachable. The layout is a Server Component that delegates client-side routing awareness to the extracted StaffNav component.
